# MediaDash Refactoring Opportunities

Analysis of potential code improvements for maintainability and performance.

## High Priority

### 1. MediaManager.swift - Extract File Operations (2785 lines)

**Location:** `MediaManager.swift:55-414` (folder/path operations)

**Issue:** `MediaManager` is a massive 2785-line file mixing UI state management with file I/O.

**Suggestion:** Extract file operations into a `FileOperationsService`:

```swift
// New: FileOperationsService.swift
@MainActor
class FileOperationsService {
    nonisolated func ensureYearFolderStructure(settings: AppSettings) throws
    nonisolated func getPaths(settings: AppSettings) -> (workPic: URL, prep: URL)
    nonisolated func prepFolderName(docket: String, date: Date, namingService: FolderNamingService) -> String
    // ... other file ops
}
```

**Benefits:**
- `MediaManager` would drop to ~1500 lines
- File operations become testable in isolation
- Easier to mock for unit tests

### 4. NotificationCenter - Extract Persistence

**Location:** `NotificationCenter.swift:1-953`

**Issue:** 953 lines combining business logic (deduplication, indexing) with persistence (UserDefaults).

**Suggestion:** Create `NotificationStore` protocol:

```swift
protocol NotificationStore {
    func load() -> [Notification]
    func save(_ notifications: [Notification])
}

class UserDefaultsNotificationStore: NotificationStore { }
```

---

### 5. Duplicate Code in Path Resolution

**Pattern seen in:** `MediaManager.swift:56-124`

Multiple functions repeat:
```swift
let serverRoot = URL(fileURLWithPath: settings.serverBasePath)
    .appendingPathComponent("\(settings.yearPrefix)\(year)")
```

**Suggestion:** Single helper:
```swift
private nonisolated func yearFolder(for year: Int) -> URL {
    URL(fileURLWithPath: settings.serverBasePath)
        .appendingPathComponent("\(settings.yearPrefix)\(year)")
}
```
